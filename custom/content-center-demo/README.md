# 整木网编辑中台 MVP

这是一个用于本地做 MVP 测试的独立版本，重点覆盖“白名单抓取、三层去重、AI 辅助、人工审核、发布归档”的演示闭环。

## 当前已覆盖的能力

- 登录与角色切换
- 抓取源管理
- 关键词管理
- 抓取任务执行
- URL / 标题 / 正文相似度三层去重
- 内容池与编辑审核台
- AI 摘要 / 标题 / 改写 / 扩写
- AI 后台配置
- 抓取失败日志
- 抓取源提取预览
- SQLite 本地持久化
- 操作日志追踪

## 当前服务端结构

目前服务端已从单个超大文件拆成 4 层：

- `src/storage/state-store.js`
  负责默认数据、SQLite 关系表、加载与写回
- `src/services/crawl-service.js`
  负责正文提取、提取预览、去重与抓取任务
- `src/services/ai-service.js`
  负责 AI 配置、提示词和 DeepSeek 调用
- `src/routes/api-router.js`
  负责鉴权、角色校验和 API 路由

`server.js` 现在只负责组装模块、静态资源服务和启动入口。

## 本轮新增重点

### 1. 服务端职责拆分

这轮不再把抓取、AI、数据库和路由全部堆在一个文件里，而是按层拆开，方便后续继续扩功能和查问题。

### 2. 状态枚举开始统一

后端已统一抽出文章状态、发布状态、去重状态、任务状态、日志类型和角色枚举，避免继续散落硬编码。

### 3. 抓取源提取预览

管理员现在可以先填写：

- `entryUrl`
- `parseRule`
- `excludeRule`

再点击“预览提取效果”，先看：

- 识别标题
- 发布时间
- 提取模式
- 提取后的正文片段

这样可以在正式跑任务前先验规则是否大致命中正文。

## 当前边界

- 真实抓取仍是最小链路，不是生产级采集器
- 正文提取规则是轻量实现，复杂站点仍可能失败
- 提取预览依赖目标网页可访问
- DeepSeek 真实调用需要你提供可用配置
- 发布回写仍是演示接口，不是正式主站发布
- 当前目录不是 Git 仓库，因此无法执行真正的 `git pull --ff-only`

## 建议测试顺序

1. 管理员登录
2. 新建或编辑抓取源
3. 先点“预览提取效果”
4. 调整 AI 配置并保存
5. 执行抓取任务
6. 查看失败日志与内容池
7. 用编辑账号做 AI 改写和提交审核
8. 用审核账号执行通过、发布、归档
