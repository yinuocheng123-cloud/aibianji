# 版本记录：v0.2-zhengmuwang-naming-and-ui-strategy

## 1. 本轮目标

本轮目标有两件事：

- 把 demo 对外名称统一改为“整木网编辑中台”
- 明确 UI 调整应该分阶段进行，而不是等所有功能做完后一次性处理

本轮目标类型：

- 功能补充
- 可维护性提升

## 2. 背景与上下文

本轮承接 `v0.1-content-center-demo-baseline`。

上一轮已经确认：

- 先用可运行 demo 表达核心主链路
- 暂不进入真实抓取、数据库、DeepSeek 正式接入
- 页面当前更偏“流程确认版”，不是最终视觉成品

用户本轮提出两个新要求：

- 名称改为“整木网编辑中台”
- 询问 UI 是不是要放到最后再调整

## 3. 问题分析

这轮的核心不是单纯改一个标题，而是确定“产品确认顺序”。

如果 UI 完全拖到最后，风险在于：

- 前面做出来的页面结构可能不符合业务使用习惯
- 信息层级、操作顺序、编辑工作台布局可能需要大改

如果现在就做最终精修 UI，风险在于：

- 真实业务流程和字段还没完全定下来
- 可能在视觉细节上投入过早，后续还要返工

因此更合理的方式不是“现在完全不调”或“现在一次性调完”，而是拆成两段。

## 4. 候选方案比较

### 方案 A：UI 完全最后再调

优点：

- 前期开发阻力小

缺点：

- 可能把结构问题拖到后面集中爆发
- 容易出现功能做完了，但页面不好用

结论：

- 不建议。

### 方案 B：现在就做最终 UI 成品

优点：

- 早期视觉完成度高

缺点：

- 功能和字段变化时返工大
- 当前阶段信息架构还在确认中

结论：

- 不建议。

### 方案 C：现在先调结构型 UI，最后再做视觉精修

优点：

- 先确认编辑效率和操作顺序
- 后续视觉优化更有依据
- 返工成本最低

缺点：

- 会经历两轮 UI 调整

结论：

- 本轮选择方案 C。

## 5. 最终决策

最终决策如下：

1. 名称现在就改，所有 demo 对外展示统一为“整木网编辑中台”。
2. UI 现在先调“结构层”和“信息层”，例如：
   - 页面命名
   - 模块顺序
   - 编辑工作台布局
   - 操作按钮优先级
   - 关键状态标识
3. UI 的最终视觉精修放到主流程和字段稳定后再做，例如：
   - 配色体系
   - 字体细节
   - 动效
   - 卡片密度
   - 更强品牌化表现

## 10. 本轮结论

最重要的结论是：

- 名称应该现在就改
- UI 不应该完全拖到最后，而应该分阶段推进

## 11. 下一轮建议

下一轮最值得做的事：

1. 先把首页、内容池、编辑工作台的文案和信息层级调成“整木网编辑中台”语境
2. 再根据你确认的业务重点，决定是否把“抓取源管理”弱化，把“待审内容池 + 编辑台”做成更强主视角
3. 等流程稳定后，再统一做品牌化视觉精修
