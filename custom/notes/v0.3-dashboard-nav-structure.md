# 版本记录：v0.3-dashboard-nav-structure

## 1. 本轮目标

本轮目标是继续推进结构型 UI 调整：

- 把仪表盘改成明确的两行布局
- 给几个核心功能加上顶部导航栏，方便快速跳转

本轮目标类型：

- 功能补充
- 可维护性提升

## 2. 背景与上下文

本轮承接 `v0.2-zhengmuwang-naming-and-ui-strategy`。

上一轮已经明确：

- 当前阶段先调结构型 UI，不做最终视觉精修
- 名称已统一为“整木网编辑中台”

用户这轮进一步明确了两个具体方向：

- 仪表盘改为两行
- 核心功能要有导航栏

## 3. 问题分析

上一版首页的问题主要有两个：

1. 仪表盘是一整排横向卡片。
   - 在桌面端虽然能放下，但视觉上偏散，重点不够集中。

2. 页面内容变多后，缺少明确导航。
   - 抓取源、关键词、内容池、编辑台、日志都在一个长页面里，首次进入不够清楚，也不方便快速定位。

这两个问题本质上都属于信息结构问题，而不是视觉细节问题，因此适合在当前阶段先处理。

## 4. 候选方案比较

### 方案 A：只改仪表盘，不加导航

优点：

- 改动最小

缺点：

- 长页面跳转问题仍然存在

结论：

- 不采用。

### 方案 B：只加导航，不改仪表盘

优点：

- 页面可快速定位

缺点：

- 仪表盘仍然像一条横向统计带，整体层次不够稳定

结论：

- 不采用。

### 方案 C：同时调整仪表盘与导航结构

优点：

- 首页结构更完整
- 页面起始区的信息层级更清晰
- 更符合“先定结构，再做精修”的节奏

缺点：

- 需要同时改 HTML 结构和样式

结论：

- 本轮采用方案 C。

## 5. 最终决策

本轮最终决策如下：

1. 在 Hero 下方增加顶部核心功能导航栏。
2. 给仪表盘增加独立区块标题。
3. 把统计卡片改为 3 列网格，使 5 个统计项自然形成两行展示。
4. 为抓取源、关键词、内容池、编辑审核台、日志增加锚点，供导航栏跳转。

## 6. 具体实现

本轮改动文件：

- `custom/content-center-demo/public/index.html`
  - 新增核心功能导航栏
  - 新增仪表盘区块容器与标题
  - 给主要模块加锚点 ID
- `custom/content-center-demo/public/styles.css`
  - 新增导航栏样式
  - 新增区块标题样式
  - 把仪表盘网格改为三列，形成两行布局

## 7. 本轮优点

- 页面入口结构更清晰
- 首屏可更快理解有哪些核心模块
- 后续即使内容继续增加，也还有基础导航可用

## 8. 本轮缺点与代价

- 当前导航还是页面内锚点导航，不是完整后台侧边导航
- 仪表盘虽然变成两行，但统计解释和筛选能力还没增强

## 9. 验证与结果

本轮验证方式：

- 检查页面结构中是否已出现导航栏和锚点
- 检查样式中仪表盘是否改为三列布局

本轮结果：

- 核心功能导航结构已加入
- 仪表盘已按两行思路改造

## 10. 本轮结论

本轮最重要的结论是：

- 先补导航和区块层次，能明显提升 demo 的可读性和可演示性

## 11. 下一轮建议

下一轮最值得做的事：

2. 再决定是否弱化抓取源与关键词在首页中的权重
3. 等结构稳定后，再统一做整木网品牌化视觉精修
