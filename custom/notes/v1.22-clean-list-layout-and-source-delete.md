# 版本记录：v1.22-clean-list-layout-and-source-delete

## 1. 本轮目标

本轮目标是继续收口整木网编辑中台 demo 的前台结构，但这次不再沿着“三列工作台”继续微调，而是直接改成更清爽的单主轴列表布局。

这轮优先处理这个问题，是因为用户给出了更明确的新要求：

- 不要继续用现在这种竖排多列的版式
- 信息源只分为关键词和抓取源
- 抓取源只保留网址和一句话简介
- 内容池只展示文章链接列表和摘要说明
- 编审台侧重修订后的文章列表
- 发布榜只展示发布后的文章列表，并按日期排序

本轮属于：

- 交互结构重整
- 前台信息架构简化
- 可测试性提升

## 2. 背景与上下文

本轮承接：

- `custom/notes/v1.21-frontstage-simplification-and-review-queue.md`
- `custom/notes/v1.20-portal-regression-and-wording.md`

上一轮已经完成的前提有：

- 顶部导航已收口为 5 个主块
- 整木网仍然只保留 3 个模拟栏目
- 内容池已经具备多选和送入编审台的动作
- 编审台已有基础编辑与发布能力

但用户在实际预览后又进一步明确了一个更关键的方向：

- 页面不能只是“模块齐全”，还要“清清爽爽”
- 需要从“工作台拼装感”切换成“列表式信息管理感”

这意味着上一轮的“三列工作台”虽然比更早版本清晰，但仍然没有到用户想要的简洁程度。

## 3. 问题分析

本轮的表面问题是“页面看起来还杂”，但真正的核心矛盾有 3 个：

1. 当前前台虽然已经按工作流拆区，但视觉上仍然像多个功能模块并排摆放
2. 信息源还保留了抓取任务等技术配置，超出了用户当前想看的最小信息
3. 关键词和抓取源虽然能保存，但还不能真正删除，不符合“可删减”的要求

进一步拆开看：

- 用户现在要的不是“更多功能”
- 而是“更少信息、更清楚结构、更直接的列表表达”

因此本轮不应该再往现有三列版式上加说明文字，而应该直接更换前台结构表达方式。

## 4. 候选方案比较

### 方案 A：保留三列布局，只删掉一些区块

核心思路：

- 保留上一轮三列布局
- 把抓取任务藏起来
- 把说明文案缩短

优点：

- 代码改动较小
- 不容易碰到现有渲染逻辑

缺点：

- 视觉上仍然是旧工作台结构
- 不能真正满足“不要用竖排版式”的要求
- 内容池、编审台、发布榜的表达方式仍然偏卡片堆叠

放弃原因：

- 这只是“删内容”，不是“换表达方式”

### 方案 B：切成单主轴区块 + 简洁列表布局

核心思路：

- 保留顶部和仪表盘
- 主体内容改成纵向区块式结构
- 每个区块内部使用清晰列表表达
- 信息源只保留关键词和抓取源
- 编审台改成“左侧修订文章列表 + 右侧在线编辑”

优点：

- 和用户描述最一致
- 页面更轻
- 内容关系更明确
- 后续继续细修时不需要推翻这一层结构

缺点：

- 需要重写前台骨架和样式
- 需要同时调整部分前端渲染函数

最终选择原因：

- 这是本轮最符合用户“清清爽爽，不要糊里糊涂”的方案

## 5. 最终决策

本轮最终采用方案 B，关键决策如下：

1. 主体区域改成单主轴区块，不再用三列工作台作为主要布局
2. 信息源只保留：
   - 关键词
   - 抓取源
3. 抓取源前台只展示：
   - 一句话简介
   - 抓取网址
   - 类型
4. 内容池用文章链接 + 摘要说明来表达，不再强调复杂状态卡片
5. 编审台左侧改为修订文章列表，右侧保留在线编辑
6. 发布榜只保留已发布文章列表，按日期倒序
7. 为了满足“可删减”，补上关键词删除和抓取源删除接口

## 6. 具体实现

### 6.1 `custom/content-center-demo/public/index.html`

这轮把前台 HTML 重新收成了简洁区块式结构。

具体改动：

- 顶部导航保留 5 个主块，但主体不再用多列工作台表达
- `#source-hub` 改成双栏简洁区：
  - 左：关键词
  - 右：抓取源
- 抓取源表单只保留前台可见字段：
  - 一句话简介
  - 类型
  - 抓取网址
- 把旧的抓取任务区保留为隐藏 DOM，不再出现在前台
- `#content-pool` 改成：
  - 搜索
  - 多选动作
  - 链接列表
- `#editor-workbench` 改成：
  - 左侧修订文章列表
  - 右侧在线编辑区
- `#publish-board` 改成：
  - 只展示已发布文章列表
  - 不再强调栏目汇总面板
- 静态资源版本号升级为：
  - `v25-clean-lists`

### 6.2 `custom/content-center-demo/public/styles.css`

本轮样式层直接从“三列工作台”切换到“清爽列表风格”。

具体改动：

- `workspace` 改成单主轴区块布局
- 新增：
  - `dual-panel`
  - `simple-panel`
  - `simple-list`
  - `article-row`
  - `workbench-grid`
- 把信息展示统一成更轻的列表项风格
- 编审台保留在线编辑能力，但布局上更像“列表 + 详情”
- 发布榜回到简单列表，不再堆额外统计卡

### 6.3 `custom/content-center-demo/public/app.js`

本轮前端逻辑主要做了 4 类调整：

1. 信息源前台简化
   - `configureFrontstageLayout()` 不再在前台暴露抓取任务
   - 把抓取源文案收成“网址 + 简介”

2. 列表渲染重写
   - `renderSources()` 改成简介 + 网址列表
   - `renderKeywords()` 改成可删减关键词列表
   - `renderArticles()` 改成文章链接 + 摘要说明列表
   - `renderReviewQueue()` 改成修订文章列表
   - `renderPublishBoard()` 改成按日期排序的已发布文章列表

3. 行为调整
   - 新增 `deriveDomainFromUrl()`，抓取源保存时自动从 URL 推导域名
   - 新增 `getWorkbenchArticles()`，把进入编审台的文章和已修订文章统一成修订列表
   - 批量动作口径统一成“发布”

4. 删减能力补齐
   - 新增 `deleteSource()`
   - 新增 `deleteKeyword()`

### 6.4 `custom/content-center-demo/src/routes/api-router.js`

本轮补了两个最小必要的后台接口：

- `POST /api/sources/delete`
- `POST /api/keywords/delete`

这样前台“可删减”不只是展示层动作，而是真能删。

## 7. 本轮优点

这轮相比上一轮的提升主要在于：

- 前台终于从“工作台拼装感”切到了“清晰列表感”
- 信息源真正只剩关键词和抓取源
- 抓取源表达方式更贴合用户预期
- 内容池更像文章链接池，而不是状态卡片堆
- 编审台和发布榜的角色更明确
- 关键词和抓取源现在都能真正删除

## 8. 本轮缺点与代价

这轮仍然有这些代价：

- 抓取任务只是从前台隐藏，并没有被彻底移除
- 前端列表变简单了，但底层仍复用原有文章状态结构
- 历史数据中已有的旧测试数据仍然可能出现在列表里
- 发布榜虽已按日期列表化，但还没有加更强的排序和筛选控制

## 10. 本轮结论

本轮最重要的结论是：

- 用户这一轮要的不是“再收一点”，而是“换一种更简洁的表达方式”
- 只要保留后端闭环不动，前台完全可以切成更清爽的列表模式
- 信息源、内容池、编审台、发布榜这四块，现在比上一轮更接近真实中台感

被证实的假设：

- 抓取源前台只保留网址和简介就足够表达
- 修订文章列表比“待编审队列”更符合当前口径
- 发布榜只保留发布日期倒序列表，更符合用户要求

## 11. 下一轮建议

1. 在正式 `3210` 页面上实测：
   - 删除关键词
   - 删除抓取源
   - 内容池送入编审台
   - 编审后发布
2. 继续清理历史数据中残留的旧乱码内容
3. 等用户确认前台结构后，再考虑是否要把“抓取任务”完全移到后台
